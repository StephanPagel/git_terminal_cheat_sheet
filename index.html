Welcome to my GIT- and TERMINAL Cheat Sheet

WAS IST GIT?!

Git ist ein Versionverwaltungstool, das bei jedem comitten eine Art Schnappschuss von deinem Projekt macht. Hinterher hat man einen Stapel von Schnappschüssen.

GITHUB vs GIT

Git und Github sind nicht das gleiche! Git ist ein VSC-Tool und GitHub ist eine Plattform, die Git benutzt. Es gibt auch andere zb. GitLab, GitBucket, BitBucket, Sourceforge etc.

WOZU BRAUCHEN WIR GIT?

Git wird überall benutzt und ist eins der wichtigsten Tools für Entwickler geworden. Dadurch können Entwickler von überall zusammenarbeiten.

WICHTIGE BEGRIFFE ZU GIT

Staged Area = Index → Daten werden für den commit vorgemerkt sind aber noch nicht hochgeladen

Terminalbefehle

git init initialisiert ein lokales git repository

git add . gibt an was alles in den commit rein soll ( . gibt an das alles in den commit sol)

git commit -m "text" hinterlegt einen message/commit ( man kann auch sagen erlegt eine version an)

git push schickt das lokale repository in das github repository

git log zeigt alle bis dato erstellten commits ausführlich an

git log --oneline zeigt alle die bis dato erstellten commits in kurzform an

git branch "name" erstellt einen neuen Branch

git branch -a zeigt alle erstellten Branches an

git branch -d "name" löscht den ausgewählten Branch

git checkout wechselt zwischen den Branches

git pull lädt die aktuelle Version des Repository von GitHub runter

git clone lädt die aktuelle Version des Repository von GitHub runter

git merge ist dafür da, um zwei Branches zusammenzuführen

GIT & VSC

A | ADDED

Ein neues file wurde zum repository hinzugefügt

M | MODIFIED

Ein bestehendes file wurde geändert

D | DELETE

Ein file wurde gelöscht

U | UNTRACKED

Das file ist neu oder wurde verändert -wurde jedoch noch nicht zum Repository hinzugefügt

C | CONFLICT

Es gibt einen Konflikt in dem file

R | RENAMED

Das file wurde umbenannt

S | SUBMODUL

Im repository existiert ein Subrepository = untergeordnetes Repository
